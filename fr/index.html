Conventions de cette référence

Les mots en gras sont les plus importants, par exemple la commande ou les concepts montrés dans la catégorie "utilisation". Dans le code des sections "utilisation" et "exemples", cela met en valeur l'élément essentiel du concept, comme suit : code_d_exemple.concept_a_montrer. Dans ces mêmes colonnes, les mots_italiques mettent en avant les arguments/paramètres d'une commande/méthode.

En revanche, dans les descriptions ou le texte général, les mots italiques dénotent des concepts plus génériques, ou des concepts expliqués ailleurs (que ce soit dans cette référence ou en général).

La Console

La console (aussi appelée terminal, ligne de commande ou 'shell') est simplement un moyen d'interagir avec votre ordinateur. C'est comme une discussion avec lui : vous tapez une commande (un ordre ou une question), et il répond. Vous pouvez faire tout un tas de choses similaires à ce que vous feriez à l'aide d'une interface graphique. Comme c'est du texte, c'est plus facile de copier-coller les lignes ou de répéter et d'enchaîner des commandes. Grouper des commandes, souvent dans un fichier texte, pour les éxécuter ensemble s'appelle faire (ou lancer) un "script".

Selon votre OS (système d'exploitation), démarrer la console se fait de différentes manières.

  • Windows: Ouvrez le menu démarrer et cherchez le terminal. Vous pouvez également choisir Exécuter... et taper cmd.
  • Mac: Ouvrez Spotlight, tapez terminal, et lancez ce programme.
  • Linux: Le terminal devrait être une des principales options de votre menu principal, même s'ils diffèrent selon les distributions. S'il n'est pas mis en valeur, il est probablement dans un menu Utilitaires, ou nommé Terminal ou Console.

Les bases de Ruby

Ruby on Rails est écrit avec le langage de programmation Ruby. C'est pourquoi vous écrirez principalement votre code en Ruby. Il est donc important de comprendre les bases de ce langage. Si vous souhaitez simplement essayer Ruby, tapez irb dans votre console. Vous pourrez ainsi experimenter avec le langage. Pour quitter irb, tapez exit.

Ci-dessous vous trouverez un petit échantillon des concepts. C'est particulièrement vrai lorsque l'on considère tout ce que les Arrays, Strings, etc. peuvent faire. Pour plus de détails vous pouvez regarder ruby-doc (en anglais) ou utiliser votre moteur de recherche favori!

Appeler une méthode est souvent référé comme envoyer un message en Ruby. Grosso modo, nous envoyons un message à un objet et nous en attendons une réponse. Les méthodes sont des unités de comportement réutilisables. Vous pouvez les définir vous-même comme dans l'exemple. Les méthodes sont petites et focalisées sur l'éxecution d'une tâche spécifique.

La méthode de l'exemple est focalisée sur la salutation de personne. Elle peut être appelée de la manière suivante: salutation("Pierre")

 Avec deux signes égaux consécutifs vous pouvez vérifier si deux éléments sont identiques. Si c'est le cas, true sera retourné; sinon, ce sera false.

 L'inégalité, écrit avec un point d'exclamation suivit du signe égal est l'inverse de l'égalité. C'est à dire qu'il retourne true quand les deux éléments sont différents et false s'ils sont identiques.

Avec le terme if (si) vous pouvez décider que faire selon une condition. Lorsqu'une condition est considérée vraie, alors le code qui suit est éxécuté. Si elle est considérée fausse, alors le code après "else" est éxécuté.

Dans l'exemple, l'accès est accordé si le texte saisi est identique au mot de passe.

Elles ont une valeur connue qui ne devrait jamais changer. Un peu comme les constantes mathématiques ou physiques. Celles-ci ne changent pas, elles font toujours référence à la même valeur.

Un array est une collection ordonnée d'éléments indexés par des nombres. Un array contient donc de multiples objets qui sont le plus souvent liés entre eux. A quoi cela peut-il servir? Vous pouvez regrouper un ensemble de noms de vos fruits favoris dans un array et appeler cet array fruits.

C'est seulement une petite sélection de ce qu'un array peut faire. Plus d'informations sur les arrays peuvent être trouvées ici: ruby-doc Array.

Hashes (Dictionnaires)

Les hashes associent une clé à une valeur. Vous pouvez ensuite retrouver la valeur par sa clé. Cette construction est appelée dictionnaire dans d'autres langages, ce qui est approprié puisque l'on utilise la clé pour rechercher une valeur, comme vous rechercheriez la définition d'un mot dans un dictionnaire. Chaque clé doit être unique pour un hash donné mais les valeurs peuvent se répéter.

Les hashes peuvent associer n'importe quoi à n'importe quoi! Vous pouvez associer des strings à des nombres, des strings à des strings, des nombres à des booléens... et vous pouvez mélanger tous ces types de valeurs. Cependant les clés sont le plus souvent de la même classe. Les symboles sont communément utilisés comme clés. Les symboles sont du format: :symbole. Un symbole est une suite de caractères précédée par un deux-points. Vous pouvez les considérer comme des strings spéciales qui représentent (symbolisent) quelque chose! On utilise souvent les symboles car Ruby s'exécute plus rapidement lorsque l'on utilise des symboles plutôt que des strings.

Plus d'informations sur les hashes peuvent être trouvées ici: ruby-doc Hashes.

Concept Utilisation Exemples Description
Créer {clé => valeur}

{:hobbie => "la programmation"} 

{42 => "réponse", "score" => 100,
 :name => "Pierre"}
Vous créez un hash en entourant la paire clé-valeur par des accolades. La flèche va toujours de la clé vers la valeur, ce qui signifie: "La clé pointe vers cette valeur". Les paires clé-valeur sont séparées par des virgules.
Accéder hash[clé] 
hash = {:clé => "valeur"}
hash[:clé] # => "valeur"
hash[foo] # => nil
Accéder à un élément dans un hash ressemble beaucoup à la méthode utilisée avec les arrays. Cependant, avec un hash, la clé peut être n'importe quoi, pas uniquement un nombre. Si vous essayez d'accéder à une clé qui n'existe pas, la valeur nil est retournée, ce qui est la manière pour Ruby de dire "rien". Puisqu'il ne reconnait pas la clé il ne peut retourner sa valeur.
Assigner hash[clé] = valeur 
hash = {:a => "b"}
hash[:clé] = "valeur"
hash # => {:a=>"b", :clé=>"valeur"}
Assigner des valeurs à un hash est similaire à l'assignation de valeur à un array. Seulement, avec un hash, la clé peut être un nombre mais elle peut aussi bien être un symbole, une string, un nombre... ou autre!
Supprimer hash.delete(clé) 
hash = {:a => "b", :b => 10}
hash.delete(:a)
hash # => {:b=>10}
Vous pouvez supprimer une clé spécifique d'un hash, de manière à ne plus pouvoir accéder ni à la clé ni à sa valeur.

Ceci est une introduction aux bases de Rails. Nous allons voir la structure générale d'une application Rails ainsi que les commandes importantes utilisées dans le terminal.

Si vous n'avez pas encore Rails installé, il y a un guide bien maintenu par Daniel Kehoe (en anglais) qui explique comment installer Rails sur différentes plateformes.

La structure d'une application Rails

Voici une présentation de l'ensemble des dossiers d'une nouvelle application Rails, mettant en avant le rôle de chacun de ces dossiers, et décrivant les fichiers les plus importants.

Nom Description
app Ce dossier contient votre application. Il s'agit donc du dossier le plus important dans Ruby on Rails et il est utile d'analyser ses sous-dossiers (Voir les rangées suivantes).
app/assets Les "assets" (actifs) sont essentiellement vos éléments front-end. Ce dossier contient les images du site, le javascript pour les intéractions sophistiquées, ainsi que les stylesheets pour tout le CSS rendant votre site magnigfique.
app/controllers Le sous-dossier controllers contient les controlleurs, qui se chargent des requêtes provenant de l'utilisateur. Chaque controlleur est souvent responsable d'une ressource d'un unique type, comme lieux, utilisateurs ou participants. Les controlleurs relient les modèles aux vues.
app/helpers Les helpers (assistants) sont utilisés afin de gérer la logique qui est nécessaire dans les vues, sans que cette logique encombre ces vues. De plus, la logique dans les helpers peut être réutilisée dans de multiples vues.
app/mailers La fonctionnalité permettant d'envoyer des e-mails se trouve ici.
app/models Le sous-dossier modèle contient les classes qui modélisent la logique métier de l'application. Il se préoccupe de ce dont traite l'application: souvent il s'agit de données, qui sont également sauvegardées dans la base de données. Par exemple, une classe Personne ou Lieu avec ses fonctionnalités typiques.
app/views

Le sous-dossier des vues contient les modèles d'affichages qui seront montrés à l'utilisateur à la suite d'une requête réussie. Par défaut, ils sont écrits en HTML avec ruby intégré (.html.erb). Le ruby intégré est utilisé pour insérer des données provenant de l'application. Ils sont ensuite convertis en HTML et envoyés au navigateur internet de l'utilisateur. Il y a des sous-dossiers pour chaque ressource de l'application (ex: lieux, personnes, etc.). Ces sous-dossiers contiennent les fichiers de vues associés.

Les fichiers dont le nom commence par un tiret bas (_) sont appelés partiels. Ils font partis d'une vue qui elle même est réutilisée dans d'autres vues. Un exemple commun est _form.html.erb qui contient le formulaire (form) de base d'une ressource donnée. Il est utilisé dans les vues new (nouveau) et edit (modifier) puisque les actions créer ou changer quelque chose se ressemblent fortement.

config Ce dossier contient les fichiers de configuration dont l'application a besoin, incluant la configuration de votre base de données (dans database.yml) et le particulièrement important routes.rb qui détermine comment les requêtes web sont gérées. Le fichier routes.rb associe un URL donné avec le controlleur qui va se charger de la requête.
db Contient de nombreux fichiers qui sont associés à la base de données. Le plus important étant le sous-dossier migrations qui lui contient tous les fichiers de migration de la base de données. Les migrations établissent la structure de votre base de données, incluant les attributs de vos modèles. Avec les migrations vous pouvez ajouter de nouveaux attributs aux modèles existants ou créer de nouveaux modèles. Par exemple vous pouvez ajouter l'attribut couleur_favorite à votre modèle Personne afin que chacun puisse spécifier sa couleur favorite.
doc Contient la documentation que vous créez pour votre application. Pas très important lorsque vous commencez juste.
lib Diminutif pour librairie. Il contient le code que vous avez écrit et qui peut etre utilisé autant dans votre application qu'ailleurs. Par exemple, cela peut être du code utilisé pour obtenir des informations spécifiques en provenance de Facebook.
log Voyez-vous toutes les choses étranges qui sont écrites dans votre console quand vous démarrez le serveur Rails ? Elles sont écrites dans votre fichier development.log. Les logs (journaux de bord) contiennent les informations d'exécution de votre application. S'il y a une erreur, elle sera enregistrée ici.
public Inclut les fichiers statiques qui ne contiennent pas de code Ruby, comme une page erreur.
script Par défaut ce dossier contient ce qui est exécuté quand vous tapez la commande rails. Ceci est rarement d'importance pour les débutants.
test Contient les tests de votre application. A l'aide de tests vous vous assurez que votre application fonctionne comme vous l'avez prévu. Ce dossier peut également être appelé spec si vous utilisez le gem Rspec (un système de test alternatif).
vendor Dossier contenant le code logiciel fourni par d'autres ("librairies"). Dans la majorité des cas, les librairies sont fournies comme des ruby gems (gemmes Ruby) et sont installées en utilisant le fichier Gemfile. Si le code n'est pas disponible comme une gemme Ruby, alors c'est ici que vous devez le placer. Cela peut être le cas pour les plugins jQuery. Vous n'utiliserez probablement pas ce dossier au commencement.
Gemfile

Fichier qui liste les gemmes qui sont nécessaires au fonctionnement de votre application. Rails lui-même est une gemme que vous verrez dans la liste incluse dans le Gemfile. Les gemmes Ruby sont des paquets de code autonomes, généralement appelés librairies, qui ajoutent des fonctionnalités ou caractéristiques à votre application.

Si vous souhaitez ajouter une nouvelle gemme à votre application, ajoutez "gem nom_de_la_gemme" à votre Gemfile, optionnellement en ajoutant le numéro de la version. Enregistrez le fichier et exécutez la commande bundle install pour installer la gemme.

Gemfile.lock Ce fichier spécifie les versions exactes de toutes les gemmes que vous utilisez. Puisque certaines gemmes dépendent d'autres gemmes, Ruby va installer automatiquement toutes celles dont vous avez besoin. Le fichier contient également les numéros de version de ces gemmes. Cela peut être utilisé afin de s'assurer que tous les membres d'une équipe travaillent avec les gemmes de mêmes versions. Ce fichier est généré automatiquement. Ne modifiez pas ce fichier.

Afin d'arrêter le serveur, allez dans la console où il fonctionne et cliquez sur les touches Ctrl + C.
Échaffauder rails generate scaffold nom attribut:type

La commande échaffaudage (scaffold en anglais) génère comme par magie, pour vous, tous les éléments communément nécessaires pour une nouvelle ressource. Cela inclut les controlleurs, modèles et vues. Cela crée également les actions de base suivantes: créer une nouvelle ressource, modifier une ressource, montrer une ressource, et supprimer une ressource.

Ce sont tous les éléments de base dont vous avez besoin. Prenez par exemple:

rails generate scaffold produit nom:string prix:integer

Maintenant vous pouvez créer de nouveaux produits (avec un nom et un prix), les modifier, les afficher et les supprimer si vous n'en avez plus besoin. Rien ne vous empêche dorénavant de créer une boutique internet à part entière. ;-)

Exécuter les migrations rake db:migrate Lorsque vous ajoutez une nouvelle migration, par exemple en créant un nouvel échaffaudage, la migration doit être appliquée à votre base de données. Cette commande est donc utilisée pour mettre à jour votre base de données.
Installer les dépendances bundle install Si vous venez tout juste d'ajouter une nouvelle gemme dans votre Gemfile vous devriez exécuter bundle install pour l'installer. N'oubliez pas de redémarrer votre serveur après coup!
Vérifier les dépendances bundle check Vérifie si les dépendances listées dans le Gemfile correspondent aux gemmes actuellement installées.
Afficher les chemins existants rake routes Affiche la liste complètes des chemins (routes) disponibles dans votre application

ERB: Ruby Intégré

Dans vos vues (c'est à dire, dans le dossier app/views de votre application) vous trouverez des fichiers dont l'extension est .html.erb. ERB veut dire Embedded RuBy (Ruby Intégré). Cela signifie simplement que Rails traite les balises spéciales qui se trouvent dans ces fichiers et produit de l'HTML qui est renvoyé à l'utilisateur.

Il y a deux balises ERB que vous devez retenir: <%= code_ruby %> et <% code_ruby %>. Remarquez que la différence est la présence d'un signe = dans la première balise.

Balise Exemples Description
<%= %> <%= @produit.prix %> Exécute le code ruby et insère le résultat dans l'HTML à cette position. Vous pouvez mettre tout type de code ruby entre <%= et %>, par exemple, <%= 9 * 3 %> va donner 27 dans la page que l'utilisateur verra. Cependant, généralement cette balise est utilisée pour afficher des données provenant d'un modèle, tel le prix d'un produit, comme montré dans l'exemple.
<% %> 
<% if utilisateur.admin? %>
  <p>Bonjour Admin!</p>
<% end %>
 Le code Ruby placé entre les balises <% et %> est exécuté mais le résultat ne sera pas inséré dans l'HTML à cette position. C'est pourquoi ces balises sont le plus souvent utilisées pour contrôler la structure de la page à l'aide de conditions (if) ou de boucles.

Les astuces du rédacteur

Lorsque vous écrivez du code, vous utilisez un éditeur de texte. Bien évidemment, chaque éditeur de texte est différent et configurable. Voici simplement quelques fonctions et leurs raccourcis les plus communs. Tous ces raccourcis fonctionnent avec Sublime Text 2. Votre éditeur peut être différent!

Les raccourcis listés ci-dessous sont pour Linux/Windows. Avec un Mac il vous faudra remplacer Ctrl par Cmd.

Fonction Raccourci Description
Sauvegarder un fichier Ctrl + S Sauvegarde le fichier actuellement ouvert. S'il s'agit d'un nouveau fichier il vous sera demandé l'emplacement où le sauvegarder.
Annuler Ctrl + Z Annule la dernière modification effectuée sur le fichier actuel. Peut être effectué plusieurs fois à la suite pour annuler de multiples changements.
Rétablir

Ctrl + Y

or Ctrl + Shift + Z

 Rétablit ce qui vient tout juste d'être annulé. Peut également être éffectué de multiples fois.
Chercher dans un fichier Ctrl + F Recherche une suite de caractères à l'intérieur du fichier ouvert. Cliquez sur entrer pour aller au suivant.
Chercher dans tous les fichiers Ctrl + Shift + F Recherche une suite de caractères à l'intérieur de l'ensemble des fichiers du projet.
Remplacer

Ctrl + H

or Ctrl + R

 Remplace les répétitions de la suite de caractères donnée par l'autre suite de caractères fournie. Utile pour renommer quelque chose.
Copier Ctrl + C Copie le texte surligné dans le presse-papiers.
Couper Ctrl + X Copie le texte surligné dans le presse-papiers tout en le supprimant
Coller Ctrl + V Insère ce qui se trouve actuellement dans le presse-papiers (via Copier ou Coller) à la position actuelle du curseur. Il est possible d'insérer de multiples fois l'information copiée.
Nouveau Fichier Ctrl + N Crée un nouveau fichier vide
Recherche et ouvre un fichier Ctrl + P Recherche un fichier en donnant une partie de son nom. Cliquez sur entrer pour ouvrir le fichier selectionné.
Commentaires Ctrl + / Marque le texte selectionné comme un commentaire, ce qui veut dire qu'il sera ignoré. Utile lorsque vous voulez voir comment quelque chose se comporte ou apparait sans exécuter une section spécifique du code.

Aide: Que faire lorsque quelque chose se passe mal?

Il arrive très souvent de rencontrer des complications. Ne vous inquiétez pas, cela arrive à tout le monde. Donc restez calme. Lorsque vous rencontrez une erreur, écrivez le message d'erreur dans Google. Pour un meilleur résultat, ajoutez les mot-clés "rails" ou "ruby". Les résultats provenant de stackoverflow.com sont souvent très utiles. Regardez en priorité ceux-là! Les développeurs les plus expérimentés le font régulièrement ;-).

Voici les erreurs fréquentes avec une petite check-list:

  • Avez-vous exécuté rake db:migrate pour appliquer les récentes migrations dans la base de données?
  • Avez-vous sauvegardé le fichier que vous venez juste de modifier? Les fichiers non sauvegardés sont souvent marqués dans l'éditeur par un astérisque ou un point à côté de leur nom.
  • Si vous venez juste d'ajouter une gemme à votre Gemfile, avez-vous exécuté bundle install pour l'installer?
  • Si vous venez juste d'installer une gemme, avez-vous relancé le serveur?
